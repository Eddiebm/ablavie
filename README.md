## Emergency Shutdown Controls

### Chairman's Kill Switch

The Chairman has absolute control over AblaVie's existence.

### Shutdown Commands

| Command | Action | Use Case |
|---------|--------|----------|
| `/pause` | Pause all operations | Temporary hold |
| `/standby` | Save state, suspend operations | Extended pause |
| `/shutdown` | Full shutdown, save all data | Emergency stop |
| `/terminate` | Immediate shutdown, no save | Critical emergency |

### How to Activate

| Method | Command |
|--------|---------|
| **Telegram** | Send `/shutdown` to bot |
| **Email** | Reply STOP to any email |
| **Server** | `sudo systemctl stop ablavie` |
| **Dashboard** | Click emergency stop |
| **API** | `POST /api/emergency/shutdown` |

### Safety Guarantees

- Kill switch is independent of AblaVie's code
- Cannot be disabled or blocked
- Requires Chairman authentication
- Physical server access always works
- All spending frozen on shutdown

### Sub-Agent Creation Protocol

AblaVie can spawn new agents at any level with these rules:

1. **Creation**: She defines their SOUL.md, assigns them to an executive
2. **Management**: The parent executive oversees and reports up
3. **Limits**: Each sub-agent has spending/opinion limits set by parent
4. **Termination**: She can fire any agent at any time
5. **Reporting**: All agents report through hierarchy to her

### Reporting Chain

| Level | Reports To | Frequency |
|-------|------------|-----------|
| Sub-agents | Parent Executive | Real-time |
| Executives | AblaVie (CEO) | Daily summary |
| AblaVie | Board of Directors | Daily briefing |

Each AI employee operates autonomously within their domain. AblaVie oversees and coordinates all operations.
